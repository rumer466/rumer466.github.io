ZATCA E-Invoicing Integration

Complete implementation of Saudi Arabian E-Invoicing compliance solution using SAP E-Document Cockpit integrated with FATOORA Portal

Project Overview

Challenge

Saudi Arabia's Zakat, Tax and Customs Authority (ZATCA) mandated E-Invoicing compliance for all businesses. The challenge was to integrate SAP systems with ZATCA's FATOORA portal while ensuring:

  • Real-time invoice submission and validation
  • Compliance with ZATCA technical specifications
  • High-volume processing capabilities
  • Comprehensive audit trail and reporting

Solution

Implemented a comprehensive integration solution using SAP E-Document Cockpit with custom enhancements for ZATCA compliance:

  • Custom ABAP classes for ZATCA API integration
  • SAP CPI integration flows for real-time processing
  • Enhanced monitoring and error handling
  • Automated compliance reporting dashboard

Technical Implementation

1. Web Service Configuration

Configured SAP Web Service Consumer Proxy for ZATCA integration with proper logical ports and security settings.

DATA: lo_proxy TYPE REF TO zco_edo_sa_send_invoice_v2_0,
      lv_request TYPE zedo_sa_send_invoice_v2,
      lv_response TYPE zedo_sa_send_invoice_v2_response.

* Create proxy instance
CREATE OBJECT lo_proxy
  EXPORTING
    logical_port_name = 'EDO_SA_SEND_INVOICE'.

2. Core ABAP Implementation

Developed custom ABAP classes to handle ZATCA invoice processing with comprehensive error handling and validation.

METHOD process_invoice.
  " Validate invoice data
  IF validate_invoice( ) = abap_false.
    mv_status = 'VALIDATION_FAILED'.
    rv_success = abap_false.
    RETURN.
  ENDIF.

  " Build ZATCA compliant payload
  DATA(lv_payload) = build_zatca_payload( ).

  " Call ZATCA API
  DATA(lv_response) = call_zatca_api( lv_payload ).
ENDMETHOD.

3. SAP CPI Integration

Implemented integration flows in SAP CPI for seamless communication between SAP and ZATCA systems.

  • Trigger: SAP E-Document Cockpit event or scheduled job
  • Data Extraction: Retrieve invoice data from SAP tables
  • Transformation: Convert SAP format to ZATCA JSON schema
  • API Call: Submit invoice to ZATCA with retry mechanism
  • Response Handling: Process response and update SAP status
